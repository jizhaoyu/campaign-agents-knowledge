# 需求清单

## 功能需求

### FR-001: MUST - 管理员可以创建知识库

验收标准：

- 可创建知识库名称、描述、状态
- 支持启用和停用

### FR-002: MUST - 管理员可以上传知识文档

验收标准：

- 支持 PDF、Markdown、TXT
- 记录上传人、时间、文件大小、解析状态

### FR-003: MUST - 系统可以对文档切片并建立索引

验收标准：

- 切片状态可追踪
- 索引失败可重试

### FR-004: MUST - 用户可以发起问答请求

验收标准：

- 仅检索用户有权访问的知识库
- 返回答案时附带引用片段

### FR-005: MUST - 系统在检索不足时进行拒答或降级提示

验收标准：

- 不返回伪确定性回答
- 拒答时说明原因

### FR-006: MUST - 系统可以检索相似历史工单

验收标准：

- 返回相似度结果和关键匹配字段

### FR-007: MUST - 系统可以生成工单草稿

验收标准：

- 自动生成标题、描述、优先级
- 用户可以手动修改

### FR-008: MUST - 高风险动作需要审批

验收标准：

- 支持待审批、已通过、已驳回状态
- 审批前不得执行最终动作

### FR-009: MUST - 系统必须记录审计日志

验收标准：

- 记录问题、引用、回答、工具调用、审批动作

### FR-010: SHOULD - 管理员可以查看基础指标

验收标准：

- 可查看问答次数、平均耗时、工具调用成功率

## 非功能需求

### NFR-001: MUST - 问答结果必须包含引用来源

### NFR-002: MUST - 高风险动作必须可追溯到审批人

### NFR-003: MUST - API 访问必须经过认证

### NFR-004: MUST - 系统必须具备基础 RBAC 能力

### NFR-005: MUST - 审计日志不可被普通用户修改

### NFR-006: SHOULD - 常规问答接口在 95 分位下响应时间不超过 5 秒

### NFR-007: SHOULD - 文档导入任务支持失败重试

### NFR-008: SHOULD - 系统支持通过配置切换模型与嵌入服务

## 约束

- 第一版为单体架构
- 不新增重型基础设施依赖
- 不默认自动执行高风险动作
