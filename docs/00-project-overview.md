# 项目概览

## 项目名称

企业知识库与工单协同 Agent 平台

## 问题定义

企业内部常见问题包括：

- 制度、产品、排障知识分散，检索成本高
- 一线员工提问后，答案难以追溯来源
- 同类问题重复开单，历史经验无法复用
- 高风险动作需要审批，但普通智能助手通常缺乏流程控制
- 管理层需要知道系统回答了什么、做了什么、是否可审计

## 项目目标

构建一个单体 Java 后端驱动的 Agent 平台，在单个系统内完成：

- 企业文档导入与知识检索
- 带引用的问答能力
- 历史工单检索与相似案例辅助
- 工单草稿自动生成
- 高风险动作审批控制
- 全过程审计与指标采集

## 目标用户

- 普通员工：提问、查询、发起工单
- 一线支持人员：处理工单、复用知识、查看相似案例
- 主管/审批人：审批高风险动作
- 管理员：维护知识库、用户、权限、系统配置

## 核心价值

- 降低企业知识检索成本
- 缩短常见问题处理时间
- 减少重复工单与描述不完整问题
- 提升 AI 输出的可控性和可信度
- 为后续扩展到更多业务系统提供基础平台

## 非目标

第一版不追求：

- 多智能体自治编排
- 全自动执行高风险操作
- 分布式微服务架构
- 复杂 BI 报表
- 花哨前端或移动端适配

## 成功信号

- 一个标准问答场景可以稳定返回带引用答案
- 一个工单场景可以从提问走到工单草稿生成
- 一个审批场景可以完整留下操作轨迹
- 管理员可以查看核心日志和指标
