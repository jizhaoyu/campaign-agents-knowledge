# 业务流程

## 参与角色

- 管理员
- 普通用户
- 支持工程师
- 审批人
- Agent 编排器

## 流程 1：文档导入

1. 管理员创建知识库
2. 管理员上传文档
3. 系统保存原始文件到对象存储
4. 系统提取文本并切片
5. 系统生成向量并写入索引
6. 系统记录文档和切片状态

结果：

- 文档可检索
- 失败文档可重试

## 流程 2：问答检索

1. 用户提交问题
2. 系统识别用户权限和可访问知识库
3. 系统执行关键词检索和向量检索
4. 系统重排结果并筛选引用片段
5. Agent 基于片段生成答案
6. 系统返回答案、引用、置信说明

结果：

- 用户获得带引用回答
- 检索不足时明确拒答或提示信息不足

## 流程 3：相似工单辅助

1. 用户在问答后选择“查看相似工单”
2. 系统基于问题和答案检索历史工单
3. 系统返回相似工单列表及相似原因
4. 用户决定是否继续生成新工单

结果：

- 降低重复开单
- 改善问题描述质量

## 流程 4：生成工单草稿

1. 用户点击“生成工单草稿”
2. Agent 汇总问题、引用证据、历史案例
3. 系统生成标题、描述、优先级、候选负责人
4. 用户确认或修改草稿
5. 系统创建待审批或待提交记录

结果：

- 工单描述更加完整
- 工单字段更加标准化

## 流程 5：审批与执行

1. 系统判断动作是否需要审批
2. 若需要审批，则创建审批任务
3. 审批人查看草稿和证据
4. 审批人通过或驳回
5. 通过后系统执行最终动作并写入审计日志

结果：

- 高风险动作受控
- 审批链路可追溯

## 流程 6：审计回放

1. 管理员查看某次会话
2. 系统展示问题、检索片段、模型输出、工具调用、审批动作
3. 管理员定位失败节点或异常回答

结果：

- 系统行为可解释
- 便于调试与复盘
