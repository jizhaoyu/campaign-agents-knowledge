## 已完成

- 支持 MySQL 迁移与 JPA 实体静态一致性守护：测试只读取 `db/mysql` SQL 和实体注解，校验实体表/列是否被迁移脚本覆盖
- 支持初版运营指标看板：管理员可查看知识库、文档、索引队列、失败索引、待审批、高风险工单和活跃 token 会话计数
- 支持运营健康摘要：后端返回健康等级、告警数、健康摘要和建议动作，前端总览页直接展示运营待办
- 支持前端客户端路由：知识库、问答、工单、审批、用户、会话和审计拥有独立 URL，刷新和浏览器前进后退保持当前页面
- 支持前端管理入口细粒度权限：用户、token 会话和审计入口分别按 `user:admin`、`token-session:admin`、`audit:read` 显示和请求
- 支持前后端一体化构建：前端产物可通过 `mvn -Pfrontend package` 打进 Spring Boot Jar，后端提供 SPA fallback 且继续保护 `/api/**`
- 支持审计调用链回看：前端可按 traceId、事件、对象和对象 ID 查询审计日志，时间线展示 payload 并支持复制 traceId
- 支持 AI 运行配置页：管理员可查看 OpenAI-compatible chat/embedding 启用状态、baseUrl、path、model、凭证配置状态和模型 Bean 可用性，且不暴露 API key
- 支持文档列表服务端分页：文件名搜索、索引状态过滤和分页参数校验在后端生效，前端文档管理页可翻页
- 支持兼容旧版文档列表接口的数据库侧过滤：未传分页参数时仍返回数组，但关键词和索引状态不再先全量加载后内存筛选
- 支持管理端用户、token 会话和审计日志分页：前端管理面板按页加载，后端保留旧数组响应兼容已有调用
- 支持相似工单有限候选集检索和解释：按问题关键词查询候选工单后评分，返回命中关键词和相似原因，避免每次检索全表扫描
- 支持相似工单匹配逻辑单独测试：分词、按命中关键词数排序、同分按新工单优先和相似原因生成已抽到 `SimilarTicketMatcher`
- 支持细粒度权限控制：后端按权限字符串授权，登录响应返回 `permissions`，审批人不能绕过审批角色边界提交工单
- 支持审批备注模板：审批人可选择标准通过/驳回备注并二次编辑，审计记录模板来源
- 支持 DOCX 文档导入（基于 DOCX 内部 `word/document.xml` 提取正文文本）
- 支持持久化索引任务队列和失败自动重试策略（单体内置 worker，尚未拆分为独立分布式任务系统）
- 支持数据库持久化 token 会话，access/refresh token 明文不落库，refresh 过期会话自动标记 revoked
- 支持 refresh token 轮换和前端 access token 过期自动刷新
- 支持当前会话登出并吊销当前 token
- 支持登录失败计数和账号临时锁定，降低暴力破解风险
- 支持登录成功、失败、锁定和登出的认证审计
- 支持 RAG 强拒答配置，可按引用数量和最高相关性分数控制是否回答
- 支持管理员查询用户状态并解除临时登录锁定
- 支持管理员 token 会话列表、单会话吊销和按用户批量吊销
