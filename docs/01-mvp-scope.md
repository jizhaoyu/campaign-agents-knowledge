# MVP 范围

## 范围原则

- 先闭环，后扩展
- 先单体，后拆分
- 先单 Agent，后多 Agent
- 先草稿和审批，后自动执行

## In Scope

### 1. 文档管理

- 创建知识库
- 上传 PDF、Markdown、TXT 文档
- 记录文档元数据
- 文档切片与索引

### 2. 检索问答

- 用户输入问题
- 系统执行混合检索
- 系统返回答案和引用片段
- 检索不足时拒答或提示补充信息

### 3. 工单协同

- 查询相似历史工单
- 自动生成工单草稿
- 支持人工修改后提交

### 4. 审批控制

- 高风险动作进入审批
- 审批通过后才能执行最终动作
- 支持审批通过与驳回

### 5. 审计与基础观测

- 记录对话、检索命中、工具调用、审批动作
- 提供基础耗时与成功率指标

## Out Of Scope

- 多租户计费
- 多模态文档理解
- 语音输入输出
- 自动关闭工单
- 跨系统工作流编排
- 复杂前端运营后台
- Kubernetes 生产级部署

## MVP 完成标准

- 至少 1 套知识库可用
- 至少 10 份样例文档可完成导入和检索
- 至少 3 个标准问题可稳定返回引用答案
- 至少 1 个工单草稿生成链路跑通
- 至少 1 个审批流程跑通
- 至少 1 条完整审计链路可回放

## MVP 不妥协项

- 不允许无引用直接给出“确定性”回答
- 不允许高风险动作绕过审批
- 不允许无日志执行工具动作
