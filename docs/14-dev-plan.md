# 开发计划

## 开发原则

- 先打通最短业务链路
- 每一阶段都能演示
- 不做无法验证的抽象

## Phase 0: 文档与脚手架

目标：

- 明确范围和设计
- 初始化仓库结构

交付物：

- 当前文档集
- Spring Boot 项目骨架

## Phase 1: 基础能力

目标：

- 登录、RBAC、统一响应、异常处理

交付物：

- auth 模块
- 基础用户角色表

## Phase 2: 知识库导入

目标：

- 文档上传、对象存储、切片、索引

交付物：

- knowledge 模块
- 文档导入流程

## Phase 3: 检索问答

目标：

- 问答接口、引用返回、拒答策略

交付物：

- chat 模块
- RAG 最小闭环

## Phase 4: 工单协同

目标：

- 相似工单检索、工单草稿生成

交付物：

- ticket 模块
- 工单草稿接口

## Phase 5: 审批与审计

目标：

- 审批流、审计日志、指标埋点

交付物：

- approval 模块
- audit 模块

## 第一周建议任务

1. 初始化 Spring Boot 项目
2. 建立数据库基础表
3. 完成登录和 RBAC 最小实现
4. 完成文档上传接口和对象存储接入

## 第二周建议任务

1. 完成文本抽取和切片
2. 完成向量入库
3. 打通最小问答接口

## 第三周建议任务

1. 打通相似工单检索
2. 打通工单草稿生成
3. 补审计日志

## 第四周建议任务

1. 完成审批流
2. 补测试和演示脚本
3. 进行一次完整端到端联调
